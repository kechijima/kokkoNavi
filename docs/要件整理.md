# こっこナビ 要件整理

> シングルマザーを対象としたLINE連携型支援情報配信アプリ  
> 管理画面（Web）＋ LINE Bot ＋ LIFF の3層構成

---

## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| サービス名 | こっこナビ |
| 対象ユーザー | ひとり親（主にシングルマザー） |
| 目的 | 支援情報・補助金・就労支援などをLINEを通じて届ける |
| 管理画面URL | https://kokkonavi.web.app |
| LINE Bot | @こっこナビ |

## 4. 管理画面 機能一覧

### ダッシュボード (`/`)
- ユーザー数・コンテンツ数・配信数のサマリー表示

### ユーザー管理 (`/users`)
- LINEユーザー一覧表示（名前・アイコン・タグ・オンボーディング状態）
- ユーザー詳細 (`/users/[id]`)：属性・タグ確認、タグの手動追加（マスタータグから選択）

### チャット (`/chat`)
- LINEユーザーとの1対1メッセージ管理
- 管理者がメッセージを入力→送信すると Cloud Functions 経由で LINE Push API で配信

### コンテンツ管理 (`/contents`)
- 支援情報コンテンツのCRUD（タイトル・本文・カテゴリ・タグ・画像・リンクURL・ステータス）
- **タグ**：タグマスターから選択（フリーテキスト入力なし）
- **公開URL自動生成**：リンクURLが空の場合、保存時に `https://kokkonavi.web.app/p/{docId}` を自動セット
- ステータス：`draft`（下書き）/ `published`（公開）

### セグメント管理 (`/segments`)
- ユーザーのタグ条件（AND/OR）でグループを定義
- ユーザー数をリアルタイム計算（Firestore クエリ）
- マスタータグから条件タグを選択
- セグメントへの一括タグ設定（arrayUnion/arrayRemove）

### 配信管理 (`/broadcasts`)
- セグメント × コンテンツ の組み合わせで配信作成
- 配信タイミング：即時 / 日時指定（scheduledAt）
- ステータス：`scheduled` → `sending` → `done` / `failed`
- 予定配信の編集・取消し機能
- Cloud Functions `broadcastScheduler` が5分ごとに実行し、予定時刻を過ぎた配信を自動送信

### 設定

#### タグ管理 (`/settings/tags`)
- タグマスターのCRUD（name, description）
- コンテンツ・セグメント・オンボーディングの各タグ選択UIの元データ

#### FAQ管理 (`/settings/faq`)
- LINEの「よくある質問」ボタンで表示されるFAQのCRUD
- 表示順の変更（▲▼）、有効/無効切り替え

#### オンボーディング設定 (`/settings/onboarding`)
- フローのステップ管理（質問文・回答タイプ・選択肢・タグマッピング）
- 回答タイプ：`single`（単一選択）/ `multi`（複数選択）/ `text`（自由記述）/ `number`（数値）
- タグマッピング：選択肢 → タグの自動付与設定（マスタータグから選択）

#### 管理者設定 (`/settings/admins`)
- 管理者の追加・ロール変更（スタッフ / オーナー）
- 新規追加はFirebase AuthでUIDを作成後、管理画面から登録

## 5. LINE Bot 機能

### リッチメニュー（5ボタン 3+2レイアウト）

```
┌──────────────────────────┬──────────────┐
│  🔍 支援情報を探す        │ 🌐 公式Web   │
│      (上段左 2/3幅)       │   サイト     │
├──────────────┬───────────┴──────────────┤
│ ❓ 質問・相談 │ 👤 プロフィール変更 │ FAQ │
└──────────────┴────────────────────┴─────┘
```

| ボタン | アクション | 動作 |
|---|---|---|
| 支援情報を探す | postback: `action=search` | カテゴリ選択クイックリプライを返信 |
| 公式Webサイト | postback: `action=website` | 「絶賛制作中」テキストを返信 |
| 質問・相談 | postback: `action=consult` | 相談受付メッセージ＋Firestoreに記録 |
| プロフィール変更 | uri: LIFF Profile URL | プロフィール変更LIFFを開く |
| よくある質問 | postback: `action=faq` | FAQ一覧をFlex Messageで返信 |

### Webhook イベント処理

| イベント | 処理 |
|---|---|
| `follow` | ユーザー登録（Firestore）→ ウェルカムメッセージ＋オンボーディングLIFFボタン |
| `unfollow` | ユーザーの `blocked: true` を設定 |
| `message` | オンボーディング中→次の質問送信 / 「よくある質問」キーワード→FAQ返信 / それ以外→管理画面で対応 |
| `postback` | リッチメニューボタンの各アクション処理 |

### カテゴリ検索（支援情報を探す）

1. カテゴリ選択クイックリプライ表示（子育て支援・住居支援・就労支援・経済支援・法律・権利・その他）
2. 選択されたカテゴリで `contents` コレクションを検索（published のみ）
3. Flex Message Carousel で最大5件表示（タイトル・60字プレビュー・「全文を読む」ボタン）
4. `linkUrl` 未設定の場合は `/p/{id}` を自動使用

### チャット返信（管理者→ユーザー）

- 管理画面で管理者がメッセージ入力 → `conversations/{userId}/messages` に `type: 'admin'` で保存
- Cloud Functions `onNewAdminMessage` が Firestore Trigger で検知 → LINE Push API で送信

## 6. LIFF ページ

### オンボーディング (`/liff/onboarding`)
- LIFF ID: `2005378903-vm7jt4ke`
- フォロー時にウェルカムメッセージから誘導
- オンボーディングフローの質問に順番に回答
- 完了すると `onboardingStatus: 'completed'` に更新
- 既に完了済みの場合は完了画面を表示

### プロフィール変更 (`/liff/profile`)
- LIFF ID: `2005378903-LXWyy1H1`
- リッチメニューの「プロフィール変更」ボタンから起動
- オンボーディングフローの各項目を個別に変更可能
- 現在値を表示し、選択式・テキスト入力で編集
- 保存時に `attributes` と `tags` を更新

### 公開コンテンツページ (`/p/[id]`)
- 認証不要（LINE外ブラウザからもアクセス可）
- `status: 'published'` のコンテンツのみ表示
- カテゴリ・タイトル・本文・タグ・画像を表示
- LINE Flex Messageの「全文を読む」ボタンから遷移

## 7. Firestore コレクション設計

### `users/{lineUserId}`
```
displayName: string          // LINE表示名
pictureUrl: string           // LINEアイコンURL
tags: string[]               // タグ一覧（タグマスターのname）
attributes: {                // オンボーディング回答
  [attributeKey]: string | string[]
}
onboardingStatus: 'in_progress' | 'completed'
onboardingStep: number       // 完了したステップ数
blocked: boolean             // ブロック済み
createdAt: Timestamp
updatedAt: Timestamp
```

### `conversations/{lineUserId}/messages/{messageId}`
```
text: string
type: 'user' | 'admin'
timestamp: Timestamp
sent: boolean                // admin送信時のLINE Push完了フラグ
sentAt: Timestamp
```

### `contents/{contentId}`
```
title: string
body: string
category: string             // '子育て支援'|'住居支援'|'就労支援'|'経済支援'|'法律・権利'|'その他'
status: 'draft' | 'published'
linkUrl: string              // 公開URL（空の場合は /p/{id} を自動生成）
imageUrl: string
tags: string[]
createdAt: Timestamp
updatedAt: Timestamp
```

### `segments/{segmentId}`
```
name: string
description: string
conditions: {
  type: 'AND' | 'OR'
  tags: string[]
}
userCount: number            // キャッシュされたユーザー数
createdAt: Timestamp
updatedAt: Timestamp
```

### `broadcasts/{broadcastId}`
```
title: string
message: string              // 添付テキスト（省略可）
contentId: string            // contentsのドキュメントID
segmentId: string            // segmentsのドキュメントID
status: 'scheduled' | 'sending' | 'done' | 'failed' | 'cancelled'
scheduledAt: Timestamp       // 配信予定日時
sentAt: Timestamp
stats: { sent: number }      // 送信数
createdAt: Timestamp
updatedAt: Timestamp
```

### `onboarding_flows/{flowId}`
```
isActive: boolean
steps: [
  {
    question: string
    type: 'single' | 'multi' | 'text' | 'number'
    options: string[]          // single/multi の場合
    attributeKey: string       // usersのattributesに保存するキー
    tagMapping: {              // 回答 → タグの対応表
      [answer]: string
    }
    placeholder: string        // text/number の場合
  }
]
```

### `tags/{tagId}`
```
name: string
description: string
createdAt: Timestamp
updatedAt: Timestamp
```

### `faqs/{faqId}`
```
question: string
answer: string
isActive: boolean
order: number
createdAt: Timestamp
updatedAt: Timestamp
```

### `admins/{uid}`
```
name: string
email: string
role: 'owner' | 'staff'
createdAt: Timestamp
updatedAt: Timestamp
```

## 8. Cloud Functions

| 関数名 | トリガー | 処理内容 |
|---|---|---|
| `lineWebhook` | HTTPS POST | LINE Webhookの全イベント処理 |
| `broadcastScheduler` | PubSub 5分毎 | scheduledAt を過ぎた配信を自動実行 |
| `onNewAdminMessage` | Firestore Create | 管理者メッセージをLINE Push APIで送信 |

全関数リージョン: `asia-northeast1`

## 12. CI/CD（GitHub Actions）

ファイル: `.github/workflows/deploy.yml`

**トリガー**: `main` ブランチへのpush

**ステップ**:
1. Node.js 20 セットアップ
2. `npm ci`（ルート）
3. `cd functions && npm ci`
4. `cd functions && npm run build`（TypeScriptコンパイル）
5. `npm run generate`（Nuxt 静的ビルド）
6. `firebase deploy --project kokkonavi`（全リソース）

**必要なGitHub Secrets**:

| Secret | 内容 |
|---|---|
| `FIREBASE_TOKEN` | `firebase login:ci` で取得 |
| `FIREBASE_API_KEY` | Firebase プロジェクトのAPIキー |
| `FIREBASE_AUTH_DOMAIN` | kokkonavi.firebaseapp.com |
| `FIREBASE_PROJECT_ID` | kokkonavi |
| `FIREBASE_STORAGE_BUCKET` | kokkonavi.firebasestorage.app |
| `FIREBASE_MESSAGING_SENDER_ID` | Firebase コンソールで確認 |
| `FIREBASE_APP_ID` | Firebase コンソールで確認 |
| `LIFF_ID` | オンボーディングLIFF ID |
| `LIFF_PROFILE_ID` | プロフィール変更LIFF ID |

## 13. ローカル開発

```bash
# 依存関係インストール
npm install
cd functions && npm install && cd ..

# 開発サーバー起動（npm を使用すること・yarnは不可）
npm run dev
# → http://localhost:3000

# 静的ビルド
npm run generate

# Firebase デプロイ（全リソース）
firebase deploy --project kokkonavi

# 個別デプロイ
firebase deploy --only "hosting" --project kokkonavi
firebase deploy --only functions --project kokkonavi
firebase deploy --only "firestore:rules,firestore:indexes" --project kokkonavi
```

> ⚠️ **注意**: `yarn dev` は使用しないこと。`npm run dev` を使用する。
